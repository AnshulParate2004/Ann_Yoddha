Detailed UX prompt for prediction results
When implementing the mobile app and web app prediction flow, remember that the end user is not using a generic AI tool. They are using a crop diagnosis product. The UI should present the prediction as a clear agricultural diagnosis and recommended action, not as raw model output.

User-facing prediction experience requirements
- After image capture or upload, show a clear loading state such as:
  - Analyzing crop image...
  - Checking for wheat disease...
- On success, show a result card or result section with:
  - disease name
  - confidence percentage
  - treatment or recommended action
  - scan timestamp
  - sync state if relevant
- If the disease is healthy, show a reassuring healthy state and preventive guidance
- If confidence is below 0.5, show uncertain instead of a disease label
- For uncertain results, guide the user to retake the image in better light with a clearer crop focus
- Always save the result into local history on mobile, even if the backend is unavailable

What the user should see after a prediction
- Disease: Yellow Rust
- Confidence: 87%
- Treatment: Apply the mapped actionable treatment text
- Status: Saved to cloud or Saved offline, sync pending

What the user should see for healthy output
- Disease: Healthy
- Confidence: 92%
- Treatment: No urgent treatment needed. Continue scouting and preventive care.

What the user should see for uncertain output
- Result: Uncertain
- Confidence: 41%
- Guidance: Retake the image in better lighting and focus on one leaf or wheat head.
- Advice: Avoid spraying solely based on this result until a clearer scan is available.

What should not be shown in normal UI
- raw class index
- logits
- tensor output
- internal backend ids
- model file names
- technical exception traces

Recommended UI actions after showing a result
- Scan Again
- View History
- Sync Now if offline data is pending
- Back to Dashboard

Recommended mobile history fields
- thumbnail image
- disease name
- confidence
- date/time
- synced or local-only badge

Recommended web diagnosis page behavior
- upload image
- preview selected image
- show loading state
- show diagnosis result block
- show treatment advice block
- optionally show confidence and status badge

Implementation guidance
- Keep the UI simple and farmer-friendly
- Prefer plain labels like Disease, Confidence, Treatment, and Status
- Avoid exposing model-oriented terminology in the main user flow
- Treat the prediction output as a diagnosis summary plus next-step recommendation
